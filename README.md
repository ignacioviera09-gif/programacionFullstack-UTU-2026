# Programación Full Stack - UTU 2026

Repositorio de contenidos y proyectos para el programa de Programación Full Stack de la Universidad Tecnológica del Uruguay (UTU) correspondiente al año 2026.

---

## 📁 Estructura del Repositorio

El repositorio se organiza en **10 módulos temáticos** (carpeta1 a carpeta10), cada uno representando un tema o unidad de aprendizaje diferente del programa:

```
programacionFullstack-UTU-2026/
├── carpeta1/        → Módulo 1
├── carpeta2/        → Módulo 2
├── carpeta3/        → Módulo 3
├── carpeta4/        → Módulo 4
├── carpeta5/        → Módulo 5
├── carpeta6/        → Módulo 6
├── carpeta7/        → Módulo 7
├── carpeta8/        → Módulo 8
├── carpeta9/        → Módulo 9
├── carpeta10/       → Módulo 10
└── README.md        → Este archivo
```

---

## 🎯 Propósito de cada Carpeta

Cada carpeta está destinada a contener:

- **Materiales teóricos**: Apuntes, guías y documentación
- **Ejemplos de código**: Snippets y proyectos de referencia
- **Ejercicios prácticos**: Actividades para practicar los conceptos
- **Proyectos**: Trabajos integradores del módulo
- **Recursos**: Enlaces, referencias y bibliografía

---

## 💡 Razones de esta Organización

### 1. **Modularidad**
   - Cada módulo es independiente, facilitando el aprendizaje progresivo
   - Permite trabajar en paralelo en temas relacionados

### 2. **Escalabilidad**
   - Estructura fácil de expandir si se agregan más módulos
   - Cada carpeta puede crecer sin afectar el resto

### 3. **Claridad y Navegación**
   - Organización intuitiva y jerárquica
   - Facilita encontrar rápidamente el material necesario

### 4. **Gestión con Git**
   - Permite un control de versiones limpio y ordenado
   - Facilita la colaboración entre estudiantes y docentes
   - Las carpetas vacías se mantienen con archivos `.gitKeep`

### 5. **Sigue Estándares de la Industria**
   - Estructura similar a la de proyectos profesionales
   - Prepara a los estudiantes para trabajar en repositorios reales

---

## 🚀 Cómo Usar este Repositorio

1. **Navega** a la carpeta del módulo que deseas estudiar
2. **Lee** la documentación y apuntes disponibles
3. **Examina** los ejemplos de código
4. **Realiza** los ejercicios prácticos propuestos
5. **Entrega** tus trabajos en tu rama personal (si se utiliza Git flow)
